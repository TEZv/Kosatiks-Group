## Навіщо

Замість трьох абстрактних рівнів Echo — **шість епох по одній книзі Series 1**. Кожен тиждень:

1. одна активна загадка (ротація в пулі епохи);
2. після **deep** solve — **1 літера** канонічного речення книги;
3. completion modal — **бейдж PNG** + прогрес `17/40` + рядок зібраного речення;
4. **колесо SPIN + AI** — лише автор (Google Auth, окремий трек; див. handoff у Cursor).

Загадки **без PIN** для всіх. PIN / Author? — тільки для колеса.

**~240 тижнів ≈ 4,6 року** матеріалу. Пул загадок **не видаляється** — лише росте; тиждень = `weekInEpoch % poolSize`.

### Формули (для `api/_epoch.js`)

```text
EPOCH_START_MS = Date.parse('2026-06-06T00:00:00Z')  // або локальний TZ — зафіксувати в коді
globalWeek     = floor((now - EPOCH_START_MS) / (7 * 24 * 60 * 60 * 1000)) + 1
epochIndex     = floor((globalWeek - 1) / 40) + 1   // 1..6
weekInEpoch    = ((globalWeek - 1) % 40) + 1        // 1..40
letterIndex    = weekInEpoch                         // літера N після deep у тижні N
```

## Правила літер

| Правило | Деталь |
|---------|--------|
| Коли видається | лише `ok: 'deep'` (не surface / wrong) |
| Скільки на тиждень | **1** літера з позиції `weekInEpoch` |
| Повторний deep | літера **не дублюється** (localStorage `klife-epoch-{id}-letters`) |
| Пробіли / пунктуація | у «збірці» показуються; у лічильнику — **лише літери** (40) |
| UA / EN | **два окремі речення по 40 літер**, змістово парні; не транслитерація |
| Статус речення | `draft` → `confirmed` (автор) → `locked` (після старту епохи) |
